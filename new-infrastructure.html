基礎設施 (Infrastructure)

MySQL 備份基礎設施架構圖

MySQL 備份策略
1.DELL EMC 使用硬體快照。
2.使用 MySQL 原生 Master-Slave 架構。
3.使用 Percona XtraBackup 進行差異備份。

MySQL 備份雲端設施架構圖

MySQL 雲端備份策略
1.使用 MySQL 的 Master-Slave 架構，將 Slave 透過 VPN 同步到 AWS EC2 主機A，主機A 再同步到 EC2 主機B。
2.主機B 使用 Percona XtraBackup 進行差異備份到 AWS EFS 長期保存。
3.搭配高效 TiDB 將讀寫分離。

雲上使用 Xtrabackup 備份策略

使用 Open source 的 Percona XtraBackup 進行自動化備份。

更版 jar 檔自動備份

更版過程中，主動備份每次更版的版本，並儲存於 AWS S3 Bucket。

Zabbix & Protheus 監控架構
